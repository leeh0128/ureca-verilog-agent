I would like you to implement a module named TopModule with the following
interface. All input and output ports are one bit unless otherwise specified.
 
- input clk
- input reset
- input in_bit
- output detected
 
The module should implement a Moore finite state machine that detects the
overlapping sequence "1011" on the input signal in_bit.
 
The FSM should monitor the in_bit input on every positive clock edge. When
the sequence 1, 0, 1, 1 has been received on consecutive clock cycles, the
output detected should go high for exactly one clock cycle (the cycle after
the final '1' of the sequence is received). The detector must handle
overlapping sequences — for example, the input stream 10111011 contains
two overlapping occurrences of 1011 and detected should pulse high twice.
 
When reset is high, the FSM should synchronously reset to its initial
(idle) state and detected should be low. Assume all sequential logic is
triggered on the positive edge of the clock.
 
Hint: You will need at least 5 states: an idle state, and states tracking
that you have seen "1", "10", "101", and "1011" so far. After detecting
the full sequence, the FSM must transition back to an appropriate state
(not idle) to allow overlapping detection.
